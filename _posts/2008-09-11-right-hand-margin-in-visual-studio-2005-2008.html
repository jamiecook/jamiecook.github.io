I don't like having lines longer than 100 characters long in my source code. Many editors will allow you to specify a right hand margin so that you can see when a line is longer and hence wrap it sensibly to the next line. To do this in Visual Studio, however, requires a bit more work. As described in this blog post, you have to edit the registry to make Visual Studio display nice dotted vertical lines at specific column:

* Open "HKEY_CURRENT_USER\Software\Microsoft\VisualStudio\9.0\Text Editor" key
* Create new string value "Guides"
* Set its value to something like "RGB(192,192,192) 119" (columns in registry are zero-based)
* Restart Visual Studio

This works in VS2005 as well, just change 9.0 to 8.0 in the reg location.
