Currently putting together a little demonstration to show off the power of the Amazon Elastic Compute Cloud (EC2) and I'm looking for an algorithm to
demonstrate that benefits well from parallelization. The requirements are that:

* We can generate input data in a reasonable time (either randomly or systematically)
* Input and output are reasonably sized (ie Less than 1Gb)
* Parallelization will occur at the machine level rather than processor or thread level
* Example will run for about 20 minutes on a single core laptop
* We are hoping to get a 10 node machine up and running and get the parallel version run in less than 5 minutes.
